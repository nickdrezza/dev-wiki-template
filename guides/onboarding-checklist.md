# New engineer onboarding checklist

**Goal: by the end of week one you can check out a repo, run it locally, and ship a small
change through review.** Work top to bottom; ask early if anything is unclear.

## Accounts & access

- [ ] Source control account added to the team's organization.
- [ ] Added to the team chat space and on-call calendar.
- [ ] Access to the password manager (shared vaults for the systems you'll touch).
- [ ] Cloud console access at the right permission level.

## Local setup

- [ ] Follow [Local development setup](local-setup.md) and get one service running locally.
- [ ] Clone the active repos you'll work in; skim each repo's `README`.

## Read these first

- [ ] This wiki's [`README`](../README.md) and [`CONVENTIONS.md`](../CONVENTIONS.md) — how we document.
- [ ] [`references/engineering-practices.md`](../references/engineering-practices.md) — how we write and review code.
- [ ] [`policies/secrets-management.md`](../policies/secrets-management.md) — how we handle secrets.
- [ ] The [systems registry](../systems/README.md) — a map of what we run and where its docs live.

## First change

- [ ] Pick a small starter task, open a pull request, and get it reviewed and merged.
- [ ] Notice a gap in these docs along the way? Add a bullet to [`GAPS.md`](../GAPS.md) — that's
      your first contribution.

# Incidents

Post-mortems and investigations — what broke, what we learned, and what we changed so it
doesn't recur. Read these to learn from a past incident.

Incidents are **immutable** point-in-time history: don't rewrite one later — link forward
instead. Name files `YYYY-MM-DD-slug.md`. Write blameless; focus on systems and process, not
people.
